## Unit 1 — 计算机基础与数制

### 进位计数制三要素

| 要素 | 说明 |
|------|------|
| **基数** | 某种计数制可用的数码个数。如二进制基数为2，数码0和1 |
| **数位** | 数码在一个数中所处的位置 |
| **位权** | 以基数为底的幂，表示该位数码代表的数值大小 |

### 定点数与浮点数

**定点整数** — 小数点隐含固定在数值位最右侧
- 十进制 `25` → 即为 25
- 二进制 `1101` → 1×2³+1×2²+0×2¹+1×2⁰ = 13

**定点小数** — 小数点固定在符号位和数值最高位之间
- 二进制 `0.101` → 1×2⁻¹+0×2⁻²+1×2⁻³ = 0.625

**浮点数** — $N = M \times R^E$
- M = 尾数（有效数字），R = 基数，E = 阶码
- 例：$1234.56 = 1.23456 \times 10^3$

### 进制转换

- **任意进制 → 十进制**：按位权展开并相加
- **十进制 → 其他进制**：除以基数倒取余数

> 0 前缀表示八进制数，`11L` 表示 long 型

### 计算机基本概念

| 概念 | 要点 |
|------|------|
| 数据单位 | bit（最小），byte（基本）= 8 bit |
| ASCII | 1字符占1字节，最高位0，低7位编码 |
| 冯·诺伊曼结构 | 运算器、控制器、存储器、输入设备、输出设备 |
| CPU | 运算器 + 控制器 |
| 主存 | 存储单元管理，地址从0开始（十六进制） |
| 外存 | 暂不执行的程序存放处，需加载到内存后CPU执行 |

### 语言分类

| 语言 | 特点 |
|------|------|
| **机器语言** | 二进制代码，计算机直接识别执行 |
| **汇编语言** | 需汇编后生成二进制代码 |
| **高级语言** | 需编译/解释，C++为编译型 |

**编译链接流程：** 源代码 → 编译（生成`.obj`目标文件） → 链接 → 可执行文件

> 编译器能检查语法错误，不能检查逻辑错误

### 算法特性

- **必须有输出**，可以没有输入
- 必须在**有限步骤后结束**，不能有死循环

## Unit 2 — 数据类型与运算符

### 基本数据类型

- `double` 型变量占 **8 字节**
- 字符串常量 `"ME"` 字符个数2，加 `\0` 共 **3 字节**

### 转义字符

以反斜杠 `\` 开头的特殊字符序列：

| 转义 | 含义 |
|------|------|
| `\n` | 换行符 |
| `\t` | 制表符 |
| `\"` | 双引号 |
| `\\` | 反斜杠本身 |
| `\'` | 单引号 |

### 运算符优先级（从高到低）

| 优先级 | 运算符 | 结合性 |
|--------|--------|--------|
| 1（最高） | `()` `[]` `.` `->` 后置`++` `--` | 左结合 |
| 2 | `!` `~` 前置`++` `--` `+` `-` `*` `&` `sizeof` | **右结合** |
| 3 | `*` `/` `%` | 左结合 |
| 4 | `+` `-` | 左结合 |
| 5 | `<<` `>>` | 左结合 |
| 6 | `<` `<=` `>` `>=` | 左结合 |
| 7 | `==` `!=` | 左结合 |
| 8 | `&`（按位与） | 左结合 |
| 9 | `^`（按位异或） | 左结合 |
| 10 | `|`（按位或） | 左结合 |
| 11 | `&&`（逻辑与） | 左结合 |
| 12 | `||`（逻辑或） | 左结合 |
| 13 | `?:`（三目） | **右结合** |
| 14 | `=` `+=` `-=` `*=` `/=` `%=` `<<=` `>>=` `&=` `^=` `|=` | **右结合** |
| 15（最低） | `,`（逗号） | 左结合 |

### 关键说明

> **逻辑非 `!`**：非零值 → true（0→1），零值 → false
> **短路求值**：`&&` 和 `||` 从左到右，结果确定后停止求值

**例题：** `int a=2; a+=a-=a*a;` 最终 `a` = ?
```
a -= a*a   → a = 2 - 4 = -2 (此时a=-2)
a += (-2)  → a = -2 + (-2) = -4
```

### 类型转换

- `'a'` 的 ASCII 为 97，`'a'-32` → 65（`char` 自动转 `int` 计算）
- 赋值后仍为 `char` 类型
- 强制类型转换 `(int)x` 不改变 x 本身的类型
- `i*f`（int×float）→ `double` 类型

## Unit 4/5 — 循环与函数

### continue vs break

| 关键字 | 作用 |
|--------|------|
| `continue` | **结束本次循环**，进入下一次循环（不是"层"） |
| `break` | **结束本层循环** |

### 数学函数

- `pow(x, y)` — 参数 x、y 均为实数
- `sqrt(x)` — 参数 x 为实数
- > 注意浮点数要写 `.0`

### 函数调用流程

1. **实参与形参结合** — 参数传递（值/引用/指针）
2. **执行函数体** — 使用形参和局部变量
3. **函数返回** — `return` 带返回值，销毁局部变量

### 函数要点

- 形参定义时必须指明数据类型和形参名
- 实参可以是变量、常量或表达式
- 原型声明可以在 `main` 内部或外部
- **结构化程序设计原则**：自顶向下、模块化、逐步求精

## Unit 6 — 数组与字符串

### 一维数组

- 元素在内存中**连续、顺序存储**
- 数组名 = 首地址（**常量**，不能自增/自减）
- 使用前必须先定义
- 是复合数据结构，**不能进行整体访问**

**静态数组**：加 `static`，未初始化则自动赋 0（`double` 型为 0.0）

### 字符串处理函数

| 函数 | 作用 |
|------|------|
| `strcat(目标, 源)` | 将源字符串连接到目标后面 |
| `strcpy(目标, 源)` | 将源字符串拷贝到目标中（覆盖原有内容） |
| `strlen(字符串)` | 返回字符串实际长度（**不包括** `\0`） |
| `strcmp(串1, 串2)` | 比较两串大小，相等返回0；串1>串2正整数；串1<串2负整数 |

> 字符数组存储的字符串**不能**直接用关系运算符比较，需用 `strcmp`
> `string` 类可以直接进行 `==` `>` `<` 比较（按 ASCII 逐字符）

### 二维字符数组

```cpp
char str[10][10];  // 可存储10个字符串，每个最长9字符（不含\0）
```

## Unit 7 — 指针

### 基本概念

- 变量的地址称为变量的**指针**
- 指针的类型 = 所指向变量的类型，不是存放的数值类型
- 指针本身占用内存空间，**指针的地址 ≠ 指针中存放的地址**

### 访问方式

| 方式 | 说明 |
|------|------|
| **直接访问** | 通过变量名存取 |
| **间接访问** | 通过地址（指针）存取 |

### 关键规则

- `&` 的操作数**必须是变量名**，不能是常量或表达式
- 指针变量和普通变量之间**不能直接赋值**
- 数组名（常量）不能作左值：`s++`、`s=s+1` 不合法
- 引用数组元素：`a[i]` `p[i]` `*(a+i)` `*(p+i)` 四种方式等价

### ++ 与 * 的优先级

`++` 和 `*` 同优先级，**右结合**：
- `++*p` → `++(*p)` — a[0] 的值加 1
- `*++p` → `*(++p)`
- `*p++` → `*(p++)`

### new 与 delete

```cpp
int *p;
p = new int(4);  // 分配存放整数的空间，初值为4
delete p;        // 释放内存
```

### 指针运算

- 指向同一数组的指针可比较（位置关系：前者↑小，后者↓大）
- 可进行**减法**（结果为元素个数）和赋值，**不能加法**
- 二维数组：`a`、`a[0]`、`*a`、`&a[0][0]` 相等

### 链表

| 操作 | 方式 |
|------|------|
| 插入 | `pNew->link = Pre->link;` → `Pre->link = pNew;` |
| 删除 | `p->next = p->next->next;` |
| 释放 | `delete` |

> 链表插入/删除**不需要移动数据物理位置**，只修改指针域

## Unit 8 — 函数进阶

### 数组作为函数参数

- 数组名作实参时自动转换为指向首元素的指针（地址传递）
- 形参 `int a[]` 本质等价于 `int *a`
- 实参为数组名，形参为指针或数组

### 函数模板

```cpp
template <class T>
T max(T a, T b) {
    return a > b ? a : b;
}
```

- 以 `template` 关键字标记开头

### 变量存储类别

| 类型 | 生命周期 | 特点 |
|------|---------|------|
| **局部自动变量** | 函数调用期间 | 每次调用重新初始化，结束释放 |
| **局部静态变量** | 整个程序运行期间 | 函数结束时仍存在，下次调用不重新赋值 |

## Unit 10 — 文件操作

### 基本概念

- 文件是操作系统数据组织的基本单位
- 存取方式：**顺序** 或 **随机**（直接）
- 二进制文件：01编码，非 ASCII，通用软件无法直接查看

### 文件操作类

| 类 | 用途 |
|----|------|
| `ifstream` | 读取文件 |
| `ofstream` | 写入文件 |
| `fstream` | 读写文件 |

> 不存在 `iofstream` 或 `outfstream`

### 常用方法

| 方法 | 作用 |
|------|------|
| `file.is_open()` | 判断文件是否打开成功 |
| `file.open()` | 关联文件并按指定方式打开 |
| `file.close()` | 关闭文件 |
| `file.eof()` | 判断文件指针是否到结尾 |
| `file.ignore(n, ch)` | 忽略 n 个字符或首次遇到 ch 后开始读取 |
| `getline(file, str)` | 读取当前行，指针移向下一行行首 |
| `>>` | 读操作（数据从外部设备到内存） |
| `<<` | 写操作（数据从内存到外部设备） |

### 输入/输出流

> C++ 中输入/输出流操作类的基类均为 `ios`
> 文件读 = 外存 → 内存，文件写 = 内存 → 外存
