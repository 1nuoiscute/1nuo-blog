随着「1nuo 评测」的上线，网站内的照片（尤其是山西之行永乐宫壁画等高清大图）越来越多，我意识到传统的“图片随源码一起存放在本地”的方式已经达到了瓶颈：GitHub 仓库日益臃肿，国内访问加载缓慢，每次部署都在痛苦转圈。

为了追求极致的访问体验，我对博客底层架构进行了一次换血 —— **全站图片正式迁移至 Cloudflare R2 工业级云存储！**

---

### 🌟 核心架构与技术升级

* **☁️ 零成本的世界级图床 (Cloudflare R2)**：放弃了国内繁琐的备案流程和的流量费，依托其遍布全球的边缘 CDN 节点，现在博客的图片加载速度实现了真正的“降维打击”。
* **🗜️ WebP 压缩**：告别了笨重的 JPG 和 PNG。通过底层插件配置，现在所有的图片在上传瞬间都会被自动转换为 WebP 格式。一张 3MB 的高清大图会被无损压缩至不到 200KB，画质不减，速度更快。
* **⚡ 全自动化写作流 (PicGo 联动)**：打通了 `截图/拖拽 -> 自动压缩 -> 云端上传 -> 返回 Markdown 链接` 的一键工作流。现在写博客，再也不用手动管理凌乱的本地图片文件夹了。
* **🪶 彻底的“图文分离”**：这也是本次升级最爽的一点！移除了本地所有的海量图片包，现在的博客源码只剩下纯粹的几兆文本。从此以后，`git push` 和 Vercel 的自动化部署全都是秒级响应！

### 🛠️ 历史包袱的“无痛清退”

面对大量历史旧图和错综复杂的 Markdown 路径，**Python 自动化清洗脚本**帮了我大忙。

仅用几秒钟，脚本就自动完成了全站旧图的 WebP 转换、云端路径正则替换，甚至顺手把「评测数据中心」里的 JSON 核心数据也完美切入到了新图床。
